# SmartRetail Domain Glossary

## Service Abbreviations

| Acronym | Full Name | Role |
|---|---|---|
| SIS | Sales Ingestion Service | Ingest POS events, deduplicate, publish to EventBridge |
| IMS | Inventory Management Service | Maintain stock levels, raise alerts |
| RE | Replenishment Engine | Purchase order lifecycle and approval state machine |
| ARS | Analytics & Reporting Service | Read-only aggregation across all schemas |
| DFS | Demand Forecasting Service | Store and serve P10/P50/P90 forecasts from SageMaker |
| SUP | Supplier Integration Service | Supplier profiles, order tracking, performance scoring |
| PPS | Pricing & Promotions Service | Promotion planning and activation (stub in prototype) |

## Supply Chain Domain Terms

**ATP** (Available-to-Promise) — `on_hand_quantity - reserved_quantity`. When ATP falls below
`reorder_point`, IMS raises a stock alert.

**DC** (Distribution Centre) — A warehouse node in the supply chain. Three DCs in seed data:
DC-LONDON, DC-MANCHESTER, DC-BIRMINGHAM.

**SKU** (Stock Keeping Unit) — A distinct product variant. 20 SKUs in seed data across Beverages,
Snacks, Dry Goods, and Dairy categories (e.g., SKU-BEV-001, SKU-SNK-002).

**PO** (Purchase Order) — A replenishment order generated by RE when a stock alert is triggered.
Goes through the DRAFT → PENDING_APPROVAL → APPROVED → DISPATCHED → SHIPPED → COMPLETED lifecycle.

**Reorder Point** — The ATP level at which a replenishment trigger fires. Stored per (sku_id, dc_id)
in `replenishment.replenishment_rules`.

**Auto-Approve Threshold** — If the PO total value is below this threshold, RE automatically
transitions from PENDING_APPROVAL to APPROVED without human review.

**Lead Time** — Days from PO APPROVED to expected SHIPPED. Used in demand forecast horizon.

**MAPE** (Mean Absolute Percentage Error) — Primary forecast accuracy KPI.
Formula: `mean(|actual - forecast| / actual) * 100`. Target < 15%.

**OTD** (On-Time Delivery) — Percentage of supplier orders delivered on or before promised date.
Target > 95% for preferred suppliers.

**Stockout** — When ATP <= 0 for a SKU at a DC. Tracked in Executive Dashboard as stockout incidents.

**Fulfilment Rate** — Percentage of demand units that could be fulfilled from on-hand inventory.

**Carrying Cost** — Cost of holding inventory (capital tied up + storage). Included in Exec Dashboard.

## Workflow Status Transitions

```
DRAFT
  |
  v  (RE generates and auto-advances, or explicit trigger from Flow 9)
PENDING_APPROVAL
  |         |
  v         v
APPROVED  REJECTED     <- planner action (RE endpoint)
  |
  v
DISPATCHED             <- supplier dispatches (SUP webhook)
  |
  v
ACKNOWLEDGED           <- DC acknowledges receipt
  |
  v
SHIPPED
  |         |
  v         v
PARTIAL_DELIVERY  COMPLETED
  |
  v
COMPLETED

CANCELLED              <- system-level only (e.g., superseded PO, expired replenishment rule)
                          NEVER used for planner rejection — that is REJECTED
```

**Rule:** A PO can only be approved from `PENDING_APPROVAL`. Calling the approve endpoint with
any other status returns HTTP 409 with `errorCode: INVALID_STATUS_TRANSITION`.

## Forecast Terminology

**P10 / P50 / P90** — Percentile forecast bands from SageMaker DeepAR:
- P10: 10th percentile — optimistic (low) demand scenario
- P50: 50th percentile — median demand (base case)
- P90: 90th percentile — pessimistic (high) demand scenario

**Forecast Horizon** — Days ahead the forecast covers (default: 30 days).

**Forecast Run** — A SageMaker batch transform job. Each run produces one `forecast_runs` record
(status: PENDING → IN_PROGRESS → COMPLETED / FAILED) and many `demand_forecasts` rows.

**SageMaker Trigger Lambda** — EventBridge scheduled rule fires it weekly to kick off a new
training + batch transform job on historical `sales.sales_events` data.

**Batch Post-Processor Lambda** — S3 ObjectCreated trigger. Reads SageMaker CSV output
(`sagemaker/output/{run_id}/part-*.csv`) and POSTs parsed rows to DFS.

## Event Types on EventBridge (`smartretail-events-{env}`)

| Event Type | Producer | Consumers |
|---|---|---|
| `SalesTransactionEvent` | SIS | IMS (via SQS) |
| `InventoryAlertEvent` | IMS | RE (via FIFO SQS), ARS (via Standard SQS) |
| `PurchaseOrderEvent` | RE | ARS (via Standard SQS) |

All events include: `eventType`, `eventId` (UUID), `timestamp` (ISO-8601), `version` (schema version).

## User-Facing Roles (Cognito)

| Role | Who | What they can do |
|---|---|---|
| STORE_MANAGER | Store manager at a DC | View stock alerts and dashboard for their DC only |
| SC_PLANNER | Supply chain planner | Approve/reject POs, view all forecasts, manage suppliers |
| SUPPLIER_ADMIN | External supplier contact | View own orders, update shipping status |
| ADMIN | Platform admin | All permissions across all DCs |
